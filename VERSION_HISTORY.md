# 版本历史

## v1.0-framebuffer-capture (2026-02-28)

**实现方式：** 基于 framebuffer 捕获

**功能状态：**
- ✅ 插件框架完整
- ✅ 几何节点编辑器顶部 Backdrop 按钮
- ✅ 实时更新（30 FPS）
- ✅ 背景保持正确的宽高比
- ✅ 不受节点编辑器缩放影响
- ❌ 显示效果没有光影（只有黑色轮廓）

**技术细节：**
- 使用 `GPUFrameBuffer.read_color()` 捕获 3D 视图
- 在节点编辑器背景绘制捕获的纹理
- 使用定时器实现实时更新

**已知问题：**
- Blender 4.5/5.0 的 `read_color()` API 只返回 911 bytes 而不是完整的数据（约 27 MB）
- 导致捕获的画面不完整，无法显示光影效果
- 详见 `KNOWN_ISSUES.md`

**结论：**
这个实现方式证明了在当前的 Blender Python API 中，通过 framebuffer 捕获无法获取完整的渲染数据。需要采用其他方式实现。

## v2.0-direct-draw (2026-02-28)

**实现方式：** 直接绘制 3D 场景

**功能状态：**
- ✅ 插件框架完整
- ✅ 几何节点编辑器顶部 Backdrop 按钮
- ✅ 实时更新（30 FPS）
- ✅ 直接绘制场景几何体
- ✅ 使用 3D 视图的相机视角和变换矩阵
- ✅ 深度测试
- ⚠️ 基本着色（使用材质颜色，但没有光照）

**技术细节：**
- 从 3D 视图获取 view_matrix 和 projection_matrix
- 遍历场景中的所有网格对象
- 使用 GPU 模块直接绘制几何体
- 应用对象的材质颜色
- 启用深度测试以正确处理遮挡关系

**改进点：**
- 不再依赖 framebuffer 捕获
- 直接访问场景几何数据
- 理论上可以显示完整的场景内容

**待改进：**
- 需要添加光照计算以获得更真实的渲染效果
- 可能需要优化性能（大场景时）
